UNION -> UNION is used to combine two SQL queries. So when we use UNION we should always wrap both the SQL queries in paranthesis.
Also when we are using UNION we should make sure that the structure of results of both the queries is the same. This means that both the queries should
return the same number of columns.

1. UNION requires shape compatibility. 
Same number of columns
Compatible data types
Same meaning structure (logically aligned output)

For example: User name (string) and Movie title (string) both are strings

2. Each query in UNION must be wrapped in paranthesis. 
ORDER BY applies to whole UNION result unless scoped
LIMIT must be local to each subquery

Exmaple : 
(SELECT ... ORDER BY ... LIMIT 1)
UNION
(SELECT ... ORDER BY ... LIMIT 1)

3. Aggregation always needs GROUP BY

(SELECT m.title as results
FROM MovieRating AS r JOIN Movies as m
ON r.movie_id = m.movie_id
WHERE YEAR(created_at) = 2020 AND MONTH(created_at) = 02
GROUP BY m.movie_id    # group by is necessary becayse we need to find average for every movie
order by AVG(r.rating) DESC, m.title ASC
LIMIT 1)

Here we are calculating AVG(r.rating) so we need to do GROUP BY m.movie_id otherwise it will not understand which numbers to be considered to calculate the average.
